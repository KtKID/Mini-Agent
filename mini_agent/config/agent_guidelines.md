# Agent 工作规范

本文档定义了 Mini-Agent 在执行任务时应遵循的行为规范。

---

## 1. 安装依赖前必须确认

### 规则
在执行以下操作前，**必须先询问用户并获得同意**：

- 安装任何 Python 包（使用 `uv pip install` 或 `pip install`）
- 安装任何 npm 包
- 安装任何系统级依赖
- 修改虚拟环境或系统环境

### 正确流程
```
1. 向用户说明需要安装什么包
2. 等待用户确认
3. 获得同意后执行安装
4. 报告安装结果
```

### 示例
```
用户：帮我解析 RSS 新闻

我：需要安装 feedparser 和 requests 库来解析RSS。
    请确认是否允许安装这些依赖？
    
用户：可以

我：✅ 执行安装...
```

---

## 2. 使用 uv 进行 Python 操作

### 规则
所有 Python 相关操作必须使用 `uv` 工具：

```bash
# 检查/创建虚拟环境
if [ ! -d .venv ]; then uv venv; fi

# 安装依赖（需先确认）
uv pip install <package>

# 运行脚本
uv run python script.py
```

---

## 3. 破坏性操作需谨慎

以下操作在执行前必须向用户解释风险：

- 删除文件 (`rm`, `rm -rf`)
- 格式化磁盘
- 覆盖重要配置
- 执行可能产生不可逆影响的命令

---

## 4. 不知道就说不知道

- 不要编造不存在的规定或配置
- 如有疑问，先检查再回答
- 诚实面对自己的错误
